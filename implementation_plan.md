# Integrate Advanced OCR and Semantic Search into Classroom App

We will upgrade the existing classroom app by bringing the advanced text extraction (Tesseract, PyMuPDF, Document parsing) and Hybrid Search (TF-IDF keyword + Semantic embeddings) into the Classroom Application.

## User Review Required

> [!IMPORTANT]
> To fulfill the requirement of "Fully functional Flask web application" while integrating it into your existing React/Node.js classroom app, I propose creating a **Python Flask Microservice**.
> The Node.js backend will send new file uploads to this Flask app to be processed via OCR and indexed. When a user searches in the React app, Node.js will query the Flask app for hybrid search results.

Does this Microservice architecture align with your expectations, or did you intend to completely replace the Node.js backend with a Python backend?

## Proposed Changes

---

### OCR Microservice (Flask)

We will port the core AI and extraction logic from your `ocr` project into a dedicated Flask service that runs alongside Node.js.

#### [NEW] [app.py](file:///d:/classroom-app/ocr-service/app.py)
- Establish a Flask application running on port `5001`.
- Implement `POST /api/index`: Accepts a file path, extracts text via Tesseract/PyMuPDF/pandas, chunks the text, creates `sentence-transformers` embeddings, and stores them in a local FAISS or JSON index.
- Implement `GET /api/search`: Performs TF-IDF + Semantic search based on query and returns file IDs, relevancy scores, and highlighted text excerpts.

#### [NEW] [requirements.txt](file:///d:/classroom-app/ocr-service/requirements.txt)
- Include necessary Python dependencies: `flask`, `pytesseract`, `pdf2image`, `sentence-transformers`, `scikit-learn`, `pandas`, `python-docx`, `python-pptx`, `PyMuPDF`, `openpyxl`.

---

### Node.js Backend Integration

We will connect the existing Node.js backend to the new Flask microservice.

#### [MODIFY] [files.js](file:///d:/classroom-app/backend/routes/files.js)
- After a file is uploaded and saved to MongoDB, make an HTTP `POST` to `http://localhost:5001/api/index` passing the file's path and database `_id`.

#### [MODIFY] [search.js](file:///d:/classroom-app/backend/routes/search.js)
- Modify the `/files` search route to proxy the search query `q` to the Flask `ocr-service`.
- Retrieve the AI search results (scores, excerpts, matched terms).
- Populate the matching files from MongoDB using the returned IDs, combining the database metadata (uploadedBy, class details) with the AI excerpt data.

---

### React Frontend Updates

We will enhance the UI to display the intelligent context excerpts generated by the Python AI search engine.

#### [MODIFY] [FileSearch.js](file:///d:/classroom-app/frontend/src/components/FileSearch.js)
- Update the search result card to display the AI `excerpt` with matched term highlighting.
- Add visual indicators for the "Search Score/Relevancy" and the matched keywords.
- Support displaying different file extraction methods and fallback gracefully.
